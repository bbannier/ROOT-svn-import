1 Scopes

Run the macro histogram.C. It is supposed to display a histogram, but something is wrong. Fix the macro so the histogram gets displayed.

The histogram is stored by the variable h1. h1 gets out of scope when the function returns, right after the histogram is drawn. By going out of scope it is destructed, the histogram is gone.

As the lecture describes, you will have to create the histogram on the heap for it to survive the end-of-scope. Look at the function sqroot in the code to see how to create an object on the heap.

Remember that objects on the heap are (usually) referenced by pointers (TH1F* in this case) and that you should use e.g. h1->Draw() to call a function on it (note the "arrow"!). SOLUTION!
